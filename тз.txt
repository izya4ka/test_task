Фронт
Написать простое Vue приложение используя TS для логики, Vite в качестве сборщика, vuetify в качестве библиотеки компонентов.
В нем должно быть две страницы (синглпэйдж апликейшон с переходом между страницами через вью роутер):

1.
Главная страница, на которой панелька в центре, занимающая 6/12 экрана на больших устройствах, 9/12 на средних и 12/12 на маленьких. 
Панелек закруглить края, цвет задника выбрать margin, все дела (крч css пощупай).
На панельке отображается время суток (в часовом времени пользователя).
Под временем кнопка. При нажатии на кнопку переход на другую страницу (ночью на страницу 3, днем на страницу 2).

2.
Страница с табличкой, в которую по рест ручке подгружаются данные с бэка (придумаешь сам какие, главное чтобы у них было поле is_archived). 
Над таблицей чекбокс "показывать архивные данные". Если чекбокс не выбран, из бэка должны подгружаться только данные с (is_archived = False), в противном случае - все данные.

3.
Страница с надписью "Иди спать!" по центру страницы и кнопкой возврата на главную под ней.

Бэк
Написать простенький бэк на fastapi, который хранит объекты для отображения на фронте (что за данные придумаешь, главное чтобы у них было поле is_archived). 
Данные может хранить у себя в памяти, может в БД (если реляционную БД будешь писать, используй postgres, для работы с ней sqlalchemy, alembic))
REST ручка на получения списка данных с фильтрацей по is_archived, openapi доки должны автогенериться через схемы pydantic.
Страничка с опенапи доками должна быть доступна с браузера.
Плюс ручка на создание новых данных.

Это все в одной репе желательно и в корневой ридмишке описать как разворачивать (можешь юзать докер компоуз, можешь не юзать, на твой выбор).
